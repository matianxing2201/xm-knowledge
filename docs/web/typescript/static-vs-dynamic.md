# 静态语言与动态语言

## 1、静态语言定义变量

静态语言变量在定义时要确定某种类型才能正确的为内存中分配相应类型的空间。 如`int a = 1;`

    1. 开辟一个int类型的空间
    2. a作为标识符
    3. a的类型为int
    4. 1赋值给a （将1存储到这个空间）

TS的类型定义不是为了在内存中分配相应数据类型的空间，TS的类型定义的作用是为了编译成JS代码之前进行静态类型检查

## 2、为什么TypeScript没有像Java一样的Char，String类型

TS的类型是为了最静态类型检查用，不是为了在内存中开辟相应数据类型空间用的。TS编译的本质是将ts代码转换成js代码，所以ts的基本数据类型都沿用了JS typeof返回的数据类型

## 3、为什么TypeScript没有像Java一样的整型与浮点型

1. Java的整型与浮点型是为了静态类型检查与分配相应的内存空间
2. TS中，设计整型、浮点型只能做静态类型检查
3. JS中，并没有对齐进行类型划分
4. 因为根据JS的业务场景来说，对整型与浮点型的区分并没有相应的要求

## 4、为什么TypeScript的基本数据类型定义都是小写

1. TypeScript的基本类型定义都是直接沿用JS typeof 返回的数据类型写法
2. TypeScript中也可以定义String，Number，Boolean。 只不过意义不同而已。

## 5、什么使用用显示类型定义？ 什么时候用类型推断？

类型推断是省略类型显示定义的方式，不需要显示定义类型，也会得出正确的类型， 如果再代码层面明确类型展示的目的，那么就要使用显示类型定义。在一般情况下，TypeScript可以进行明确的类型推断，就不在需要进行显示类型定义。 特别是在变量定义与初始化的时候。
