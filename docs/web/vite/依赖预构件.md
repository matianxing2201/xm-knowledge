# 依赖预构建

依赖预构建是Vite在首次启动时执行的优化步骤，它主要有两个功能

1. <font style="color:#ED740C;">它将 CJS/UMD 模块转换成 ESM 的格式</font>
2. <font style="color:#ED740C;">将多个内部模块的EMS依赖关系合并成一个</font>

## 为什么需要依赖预构件

1. CJS和UMD兼容，VIte将开发服务器所有的代码视为原生ESM，预构件的作用是将他们转换为ESM
2. 性能优化：将多个分散的模块合并为单个模块，减少网络请求（如引入lodash 有数百个模块， 预构件后 网络请求只有一个）
3. 缓存 预构件的结果都存储在`node_modules/.vite`中，后续启动直接使用

## 自定义行为

排除依赖项

```vue
export default { optimizeDeps: { exclude: ['某个不需要预构建的依赖'] } }
```

强制包含依赖项

```vue
export default { optimizeDeps: { include: ['某个未被自动发现的依赖'] } }
```

自定义预构件入口

```vue
export default { optimizeDeps: { entries: [ // 自定义扫描入口文件
'./src/main.ts', ] } }
```

## 缓存

### 文件系统缓存

预构件结果默认存储在`node_modules/.vite`中，当一下几种情况触发时会决定是否需要重新运行预构件。

1. `package-lock.json`，`yarn.lock`，`pnpm-lock.yaml`
2. 补丁文件夹的修改时间
3. `vite.config.js`中的相关字段
4. `NODE_ENV`的值

强制vite重新依赖预构件。 在启动开发服务器是指定 `--force`选项 或者手动删除`node_modules/.vite`缓存目录

### 浏览器缓存

预构件依赖的请求使用HTTP头`max-age=31536000, immutable`进行强制缓存。缓存后这些请求将不会再次访问开发服务器。
