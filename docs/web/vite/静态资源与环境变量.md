# 静态资源与环境变量

## 静态资源

### 资源目录

```vue
// vite.config.js export default { publicDir: 'public', //
默认就是public目录，可自定义 如修改后 需要创建对应的文件夹 }
```

### 各种文件的引入

```vue
// 直接使用路径引用 import logo from './assets/logo.png' function App() { return
<img src="{logo}" alt="Logo" />
}
```

```vue
// 直接导入会返回解析后的对象 import packageJson from './package.json'
console.log(packageJson.version)
```

```vue
// 使用 ?worker 后缀 import Worker from './worker.js?worker' const worker = new
Worker()
```

### 修饰符

```vue
// 获取原始URL import imgUrl from './image.png?url' // 获取原始内容 import
rawShader from './shader.glsl?raw' // 内联为base64 import inlineImg from
'./image.png?inline' // 作为worker引入 import Worker from './worker.js?worker'
```

## 环境变量

### 内置变量

```vue
console.log(import.meta.env.MODE) // (development/production)
console.log(import.meta.env.BASE_URL) // 部署的基础URL
console.log(import.meta.env.PROD) // 是否生产环境
console.log(import.meta.env.DEV) // 是否开发环境
console.log(import.meta.env.SSR) // 是否服务端渲染
```

### 模式 优先级从低到高

1. `.env` - 所有环境都会加载
2. `.env.development` - 开发环境加载
3. `.env.production` - 生产环境加载
4. `.env.development.local` - 开发环境加载，但会被git忽略(适合多人合作)

### 自定义

- `VITE_A_VAL = "自定义的值"`

### TypeScript类型支持

```vue
///
<reference types="vite/client" />

interface ImportMetaEnv { readonly VITE_A_VAL: string //
添加其他自定义环境变量... } interface ImportMeta { readonly env: ImportMetaEnv }
```
