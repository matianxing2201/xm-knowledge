# 插件机制概述

## 概述

Vite 插件是一个返回对象的函数，该对象包含特定的属性、钩子和其他配置选项。插件用于扩展 Vite 的功能，可以修改配置、转换代码、添加中间件等。

基本结构

```typescript
export default function myPlugin(options) {
  return {
    name: "my-plugin",
    // 插件配置
  };
}
```

## 属性

### name

- 插件的名称，用于错误信息和警告
- 命名规范：Vite 插件：
  - `vite-plugin-*` 或带 `vite:` 前缀 Rollup 插件：
  - `rollup-plugin-*` 或带 `@rollup/plugin-` 前缀

### enforce

控制插件执行顺序

- `pre` - 在核心插件之前执行
- 默认 - 在核心插件之后执行
- `post` - 在核心插件之后执行

### apply

制定插件的使用场景 `build`- 仅在生产构建应用 `serve` - 仅在开发环境应用。 不指定 默认两种场景都应用。

## 钩子

### 通用钩子

- options - 读取或修改 Rollup 选项
- buildStart - 开始构建时
- resolveId - 解析模块 ID
- load - 加载模块内容
- transform - 转换模块内容
- buildEnd - 构建结束时
- closeBundle - 打包结束时

### Vite 独有钩子

#### config

在解析Vite配置前调用，可以修改配置或返回配置合并

```typescript
config(config, env) {
  // 修改配置
  if (env.mode === 'development') {
    return { server: { port: 8080 } }
  }
}
```

#### configResoleved

配置解析完成后调用，可以读取最终配置

```typescript
configResolved(resolvedConfig) {
  // 存储解析后的配置
  this.config = resolvedConfig
}
```

#### configureServer

用于配置开发服务器，可以添加中间件

```typescript
configureServer(server) {
  server.middlewares.use((req, res, next) => {
    // 自定义请求处理
  })
}
```

## 开发规范

- Rollup 插件应该有一个带 `rollup-plugin-` 前缀、语义清晰的名称。
- 在 package.json 中包含 `rollup-plugin` 和 `vite-plugin` 关键字。
- Vite 插件应该有一个带 `vite-plugin-` 前缀、语义清晰的名称。
- 在 package.json 中包含 `vite-plugin` 关键字。

如果插件只适用于特定框架，它的名字应该遵守以下前缀格式

- `vite-plugin-vue-` 前缀作为 Vue 插件
- `vite-plugin-react-` 前缀作为 React 插件

## 插件顺序

- 别名插件（vite:pre-alias、@rollup/plugin）
- enforce:pre
- 核心插件（vite:resolve、vite:transform、vite:html css json）
- enforce: 未指定
- vite 构建插件
- enforce: post
- 后置构建插件
