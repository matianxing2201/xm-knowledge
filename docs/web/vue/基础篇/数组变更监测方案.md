# 数组变更监测方案

事件处理 -> 绑定事件处理函数

事件时不需要绑定的

浏览器 -> 自带了许多事件

每个事件可以绑定处理器 -> 处理函数

事件 -> 用户行为 -> 触发 -> 执行事件对应的处理函数

事件 -> 处理函数 -> 进行绑定行为

事件的触发 -> 执行其绑定的处理函数

# 1、事件监听

通过 `v-on` 指令 监听 DOM 事件，并在触发时间时执行JavaScript。 `v-on:click="methodName"` 或者快捷方式 `@click="methodName"`。

```javascript
<template>
  <div>
    <!-- 不推荐 -->
    <div>{{ count }}</div>
    <button @click="count + 1">add</button>
    <button @click="count - 1">minus</button>
  </div>
</template>

```

# 2、绑定处理函数

通常事件处理逻辑会比较复杂，所以把JavaScript代码写在 `v-on` 指令中是不推荐的

```javascript
<template>
  <div>
  	<!-- 绑定处理函数 （逻辑较为复杂） -->
    <div>{{ count }}</div>
    <button @click="addCount">add</button>
    <button @click="minusCount">minus</button> -->
  </div>
</template>
export default {
  data() {
    return {
      count: 0,
    };
  },
  methods: {
    addCount() {
      this.count += 1;
    },
    minusCount() {
      this.count -= 1;
    },
  },
};
```

# 3、内联处理器中的方法

除了直接绑定到一个方法，也可以在内联JavaScript语句中调用方法

```javascript
 <!-- 内联绑定处理函数 -->
 <!-- 调用：这里不会执行methods内对应的方法，调用的目的是为了传入实参 -->
<div>{{ count }}</div>
<button @click="addCount(1)">add</button>
<button @click="minusCount(2)">minus</button>

 <!-- 内联绑定处理函数 -->
 <!-- $event是vue封装的时间对象 变量名不可变，vue中的特殊变量 -->
<div>{{ count }}</div>
<button @click="addCount(1,$event)">add</button>
<button @click="minusCount(2,$event)">minus</button>
```

# 4、多事件处理函数绑定

可以绑定多个事件处理函数，使用 逗号运算符分隔

```javascript
<div>{{ count }}</div>
<button @click="addCount(1),setLog('log')">add</button>
<button @click="minusCount(2),setLog('log')">minus</button>
```

# 5、事件修饰符

目的在于把事件处理函数中非纯逻辑的程序分离出去

| .prevent | 事件处理函数内部 -> Event.preventDefault()                 |
| -------- | ---------------------------------------------------------- |
| .once    | 只调用一次事件处理 -> 调用一次以后自动移除监听器 once:true |
| .stop    | 阻止事件冒泡 cancleBubble stopPropagation()                |
| .capture | 采用捕获 capture:true                                      |
| .self    | 跳过非事件源的事件触发                                     |
| .passive | 永远不调用Event.preventDefault()                           |

> **Tips**
>
> 使用修饰符时，执行的顺序按照写的顺序执行
>
> 如 <font style="color:rgb(71, 101, 130);">v-on:click.prevent.self</font><font style="color:rgb(34, 34, 34);background-color:rgb(248, 248, 248);"> 会阻止所有的点击 </font>
>
> <font style="color:rgb(71, 101, 130);"> v-on:click.self.prevent</font><font style="color:rgb(34, 34, 34);background-color:rgb(248, 248, 248);"> 会阻止对元素自生的点击</font>

# 6、按键修饰符

可以将 [KeyboardEvent.key](https://developer.mozilla.org/en-US/docs/Web/API/KeyboardEvent/key/Key_Values) 暴露 的任意有效案件名转换为 kebab-case 来作为修饰符

Vue为常用的键提供了别名

- .enter
- .tab
- .delete (捕获“删除”和“退格”键)
- .esc
- .space
- .up
- .down
- .left
- .right

# 7、为什么在HTML中监听事件

Vue.js 事件处理方法和表达式都时严格绑定在当前视图的 ViewModel上。通过addEventListener 绑定事件处理函数而不是通过 on...的方式 来绑定。 视图与逻辑时分开的 使用 v-on 或者 @ 有以下好处

1. HTML模板中能够轻松定位在 JavaScript 代码里对应的方法
2. 无需手动绑定事件，ViewModel是<font style="color:#FA541C;">纯粹的逻辑</font>，和DOM<font style="color:#FA541C;">完全解耦</font>
3. 当ViewModel被销毁时，所有的事件处理函数都会<font style="color:#FA541C;">自动删除</font>。无需担心如何清理它们
