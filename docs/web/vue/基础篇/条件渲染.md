# 条件渲染 v-if v-else-if v-else v-show

# v-if

1、`v-if` 指令用于条件性的渲染一块内容。这块内容只会在指令的表达式返回 <font style="color:#FA541C;">truthy </font>值的时候被渲染。

2、因为 `v-if` 是一个指令。所以必须将它添加到一个元素上。 vue提供了 `<font style="color:#FA541C;"><template></font>` 元素当做一个不可见的包裹元素，并可以在上面使用 `v-if`。

3、`v-else` 元素<font style="color:#FA541C;">必须</font>紧跟在 `v-if` 或者 `v-esle-if` 的元素后面，<font style="color:#FA541C;">否则不会被识别</font>

```javascript
template:`
	<div>
  	<template v-if="isLogin">
  		已登录
  	</template>
		<template v-else>
      登录  之策
    </template>
  </div>
`

data(){
	return{
  	isLogin:true
  }
}
```

# v-show

1、与 `v-if` 一样，只有在指令的表达式中返回 truthy 值的时候才会被渲染

2、`v-show` 的元素<font style="color:#FA541C;">始终</font>会被渲染并保留在DOM中。`v-show` 只是通过 `<font style="color:#FA541C;">display</font>` 属性来控制显示隐藏

3、`v-show` 不支持 `<template>` 元素，也不支持 `v-else`。

# 总结

1. v-if是对DOM移除和添加，在移除的时候用注释节点占位。对内部的子组件与事件监听都会销毁与重建
2. v-if只有条件是truthy的时候，才会被渲染（惰性渲染）v-show总是会被渲染，用display来控制其显示与影藏
3. v-if在切换的时候会提高开销，如果条件为假值，初始化是不会进行的，v-show在切换的时候开销较低，但是初始化渲染时无论显示与否都要被渲染
4. 如果切换频繁就是用v-show。如果切换不频繁，加载时不需要的视图，可以用v-if
