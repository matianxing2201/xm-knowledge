# MVVM 专题

# 认识与实现MVC

| M：                                                                                                                               | Model 数据模型（模型层） -> 操作数据库（操作数据库）对数据进行增删改查的操作                                 |
| --------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| V：                                                                                                                               | View 视图层 -> 显示视图或视图模板                                                                            |
| C：                                                                                                                               | Controller 控制层 > 逻辑层 数据和视图关联挂载和基本的逻辑操作<br/> API层 前端请求的API对应的是控制器中的方法 |
| **服务端渲染 **View 需要数据 -> Controller对应的方法 ->调用Model的方法 -> 获取数据 ->返回给Controller对应的方法 -> renderView 中  |                                                                                                              |
| **前端渲染 **<br/> 异步请求URL -> 控制器中的一个方法 -> Model层的方法 -> 操作数据库 -> 获取数据 -> 返回给控制器方法 -> 响应回前端 |                                                                                                              |

## 前端MVC

| Model      | 管理视图所需要的数据 -> 数据与视图的关联 |
| ---------- | ---------------------------------------- |
| View       | HTML模板 + 视图渲染                      |
| Controller | 管理事件逻辑                             |

### 计算器例子

| Model      | data(数据) -> 数字a、数字b、操作符号(+-\*/)、结果 watch(监听)-> change(数据改变) -> update view(更新视图) |
| ---------- | --------------------------------------------------------------------------------------------------------- |
| View       | template(模板) -> render(渲染)                                                                            |
| Controller | event trigger(事件触发) -> model/data                                                                     |

Controller -> model -> view

view -> controller -> model
