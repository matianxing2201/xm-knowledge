# 插槽

插槽 slot 内容占位标签

# 1、插槽内容

Vue实现一套内容分发的API，设计灵感源于 [Web Components 规范草案](https://github.com/w3c/webcomponents/blob/gh-pages/proposals/Slots-Proposal.md)，将 `<slot>` 元素作为内容占位的标签 如下

```javascript
my-button 模板 组件

<button class='btns'>
	<slot></slot>
</button>

使用模板组件,当组件渲染时, slot会被替换

<my-button>                 <button class='btns'>
	click          ->						click
</my-button>							  </button>

```

slot可以渲染 普通文本、HTML、组件

```javascript
<todo-button>
  <!-- 添加一个Font Awesome 图标 -->
  <i class="fas fa-plus"></i>
  Add todo
</todo-button>

<todo-button>
    <!-- 添加一个图标的组件 -->
  <font-awesome-icon name="plus"></font-awesome-icon>
  Add todo
</todo-button>
```

如果 `<todo-button>` 的template中<font style="color:#FA541C;">没有</font>包含一个`slot`元素，则里面的内容会被抱起，不进行渲染

`<slot>` 中间的内容 是默认值

# 2、具名插槽

- slot 元素 定义 name 属性 可以用来定义插槽
- 不带name属性的slot 会隐式给 name 命名为default
- 向具名插槽传递内容是，可以在一个`<template>` 元素上使用 `v-slot` 指令，并以参数的形式提供名称
- v-slot 只能添加在 template 上

```javascript
<template v-slot:header>my header</template>
<template v-slot:main>my main</template>


 <header>
  <slot name="header"></slot>
</header>
<main>
    <slot name="main"></slot>
</main>
```

# 3、作用域插槽

- 插槽内部能够访问子组件中的数据
- 当提供的内容只有默认插槽时，组件的标签可以被当作模板来使用 再组件中直接使用 `v-slot`
- `v-slot` 可以通过解构的方式传入插槽

```javascript
父组件
<pic-board>
  <template v-slot:default="slotProps">
		{{ slotProps.item }}
  </template>
</pic-board>
解构v-slot
<pic-board>
  <template v-slot:default="{ item }">
		{{ item }}
  </template>
</pic-board>

子组件
<li v-for="item in todoList" :key="item.id">
  <div>
    <h1>{{ item.title }}</h1>
  	<slot :item=item></slot>
  </div>
</li>
```
