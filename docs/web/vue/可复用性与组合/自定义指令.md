# 自定义指令

## 简介

除了核心功能默认内置的指令（`<font style="color:#FA8C16;">v-model</font>` 和 `<font style="color:#FA8C16;">v-show</font>`），Vue 也允许注册自定义指令。注意，在Vue2.0中，代码复用和抽象的主要形式是组件。然而，有的情况下，任然需要对普通 DOM 元素进行底层操作，这时候就会用到自定义指令。

聚焦输入框例子如下：

当页面加载时，该元素将获得焦点。使用自定义指令来实现功能

### 全局注册

```javascript
// focus组件对象
import Vue from "vue";

const focus = {
  // 当被绑定的元素插入到 DOM 中时候
  inserted: function (el) {
    //聚焦元素
    el.focus();
  },
};

export default focus;

// main.js 中注册
import focus from "@/components/Focus/Focus";
Vue.directive("focus", focus);
```

### 局部注册

组件中接收一个 `<font style="color:#FA8C16;">directives</font>` 的选项

```javascript
directives: {
  focus: {
    // 指令的定义
    inserted: (el) => {
      el.focus();
    };
  }
}
```

然后就可以在模板中任何元素上使用新的 `<font style="color:#FA8C16;">v-focus</font>` 属性

```javascript
<input v-focus />
```

## 钩子函数

一个指令定义对象可以提供如下几个钩子函数（可选）

`<font style="color:#FA8C16;">bind</font>`：只调用一次，指令第一次绑定到元素时调用。在这里可以进行一次性的初始化设置

`<font style="color:#FA8C16;">inserted</font>`：被绑定元素插入父节点调用（仅保证父节点存在，但不一定被插入文档中）

`<font style="color:#FA8C16;">update</font>`：所在组件的 VNode 更新时调用，**但是可能发生在其子 VNode 更新之前**。指令的值可能发生了改变，也可能没有。可以通过比较更新前后的值来忽略不必要的模板更新

`<font style="color:#FA8C16;">componentUpdated</font>`：指令所在组件的 VNode **及其子 VNode** 全部更新后调用

`<font style="color:#FA8C16;">unbind</font>`：只调用一次，指令与元素解绑时调用

## 钩子函数参数

指令钩子函数会被传入一下参数

- `<font style="color:#FA8C16;">el</font>`：指令锁绑定的元素，可以用来直接操作 DOM
- `<font style="color:#FA8C16;">binding</font>`：一个对象，包含一下 property
  - `<font style="color:#FA8C16;">name</font>`：指令名，不包括 `<font style="color:#FA8C16;">v-</font>` 前缀
  - `<font style="color:#FA8C16;">value</font>`：指令的绑定值，列入：`<font style="color:#FA8C16;">v-my-directive = "1  + 1"</font>` 中，绑定值为 `<font style="color:#FA8C16;">2</font>`
  - `<font style="color:#FA8C16;">oldValue</font>`：指令绑定的前一个值，仅在 `<font style="color:#FA8C16;">update</font>` 和 `<font style="color:#FA8C16;">componentUpdated</font>` 钩子中可用，无论值是否改变都可用
  - `<font style="color:#FA8C16;">expression</font>`：字符串形式的指令表达式。例如 `<font style="color:#FA8C16;">v-my-directive = "1 + 1"</font>` 中，表达式为 `<font style="color:#FA8C16;">"1 + 1"</font>`
  - `<font style="color:#FA8C16;">arg</font>`：传给指令的参数，可选。例如 `<font style="color:#FA8C16;">v-my-directive:foo</font>` 中，参数为 `<font style="color:#FA8C16;">"foo"</font>`
  - `<font style="color:#FA8C16;">modifiers</font>`：一个包含修饰符的对象，例如 `<font style="color:#FA8C16;">v-my-directive.foo.bar</font>` 中，修饰符对象为 { foo：true，bar：true }
- `<font style="color:#FA8C16;">vnode</font>`：Vue 编译生成的虚拟节点
- `<font style="color:#FA8C16;">oldVnode</font>`：上一个虚拟节点，仅在 `<font style="color:#FA8C16;">update</font>`<font style="color:#FA8C16;"> </font>和 `<font style="color:#FA8C16;">componentUpdated</font>` 钩子中可用

> 除了 `<font style="color:#FA8C16;">el</font>`<font style="color:#FA8C16;"> </font>之外，其余其他参数都应该是只读的，切勿进行修改。如果需要在钩子之间共享数据，建议通过元素的 `<font style="color:#FA8C16;">dataset</font>`<font style="color:#FA8C16;"> </font>来进行

```javascript
export default {
  created,
  beforeMount,
  mounted,
  beforeUpdate,
  updated,
  beforeUnmount,
  Unmounted,
};

/**
 *
 * @param el  被绑定指令的元素，DOM对象  (对象) {}
 * @param bindings    四个参数
 *  arg: 指令的参数 -> myShow:abc -> abc
 *  dir: 指令对象里的所有属性
 *  instance: 使用指令的组件实例
 *  modifiers: 指令的修饰符集合 -> nyShow:abc.test   modifiers:{test:true}
 *  oldValue: 更新前的指令绑定值   只存在 beforeUpdate/updated    其余都是undefined
 *  value: 当前指令绑定的值
 * @param vnode  绑定指令的元素的虚拟节点
 * @param prevNode  上一个虚拟节点  beforeUpdate/updated 存在
 */

// 在绑定元素attr或监听事件绑定事件处理之前
function created(el, bindings, vnode, prevNode) {
  console.log("created", el);
}

// 当指令第一次绑定到元素并且在挂载父组件之前调用
function beforeMount(el, bindings, vnode, prevNode) {
  console.log("beforeMount", el);
}

// 在绑定元素的父组件被挂载前调用
function mounted(el, bindings, vnode, prevNode) {
  console.log("mounted", el);
}

// 在更新包含组件的VNode之前调用
function beforeUpdate(el, bindings, vnode, prevNode) {
  console.log("beforeUpdate", el);
}

// 在包含组件的VNode以及子组件的VNode更新后调用
function updated(el, bindings, vnode, prevNode) {
  console.log("updated", el);
}

// 卸载绑定元素的父组件之前调用
function beforeUnmount(el, bindings, vnode, prevNode) {}

// 当指令与元素解除绑定且父组件卸载,只调用一次
function Unmounted(el, bindings, vnode, prevNode) {}
```

## 动态指令参数

指令的参数可以是动态的。例如，在 `<font style="color:#FA8C16;">v-mydirective:[argument] = "value"</font>` 中，`<font style="color:#FA8C16;">argument</font>` 参数可以根据组件实例数据进行更新！这使得自定义指令可以在应用中被灵活使用。
