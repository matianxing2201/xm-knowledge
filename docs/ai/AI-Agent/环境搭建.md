# 环境搭建

## 1. 创建虚拟环境

### 为什么要虚拟环境？

```
项目 A：Flask 2.0 + openai 0.28
项目 B：Flask 3.0 + openai 1.0
                ↓
pip 全局安装 → 冲突！只能装一个版本
虚拟环境 → 每个项目隔离，互不影响
```

### 命令

```bash
# Python 3.3+ 自带 venv 模块，无需额外安装
python3 -m venv .venv
```

| 参数      | 含义                                               |
| --------- | -------------------------------------------------- |
| `python3` | 使用的 Python 解释器                               |
| `-m venv` | 以模块方式运行 venv（venv = virtual env 缩写）     |
| `.venv`   | 虚拟环境目录名（约定俗成，也可以叫 `env`、`venv`） |

### 创建后目录结构

```
api/
├── .venv/             ← 虚拟环境（所有依赖装在这里）
│   ├── bin/           ← 激活脚本、python 解释器
│   ├── lib/           ← 安装的包
│   └── pyvenv.cfg     ← 配置文件（指向系统 Python）
├── run.py
└── app/
```

## 2. 激活虚拟环境

### macOS / Linux

```bash
source .venv/bin/activate
```

### Windows

```bash
.venv\Scripts\activate
```

### 激活后有什么变化？

```
激活前：which python  → /usr/bin/python3        （系统全局）
激活后：which python  → /api/.venv/bin/python   （项目隔离）
```

终端前面会出现 `(.venv)` 提示：

```bash
(.venv) mtx@mac api %          ← 多了 (.venv)
```

### 退出虚拟环境

```bash
deactivate
```

## 3. 配置国内镜像

### 为什么要配？

```
pip 默认源：https://pypi.org/simple      （服务器在美国，慢！）
国内镜像：  https://mirrors.aliyun.com    （服务器在国内，快！）

实测对比：
  默认源安装 Flask：30-60 秒
  阿里云镜像安装 Flask：3-5 秒
```

### 永久配置（推荐）

```bash
# 阿里云镜像（最稳定）
pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/

# 清华大学镜像（备选）
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple/

# 中国科技大学镜像（备选）
pip config set global.index-url https://pypi.mirrors.ustc.edu.cn/simple/
```

配置后写入 `~/.pip/pip.conf`（macOS/Linux）或 `%APPDATA%\pip\pip.ini`（Windows），**一次配置，全局生效**。

### 临时使用（仅本次）

```bash
pip install flask -i https://mirrors.aliyun.com/pypi/simple/
```

### 验证镜像是否生效

```bash
pip config list
# 输出：global.index-url='https://mirrors.aliyun.com/pypi/simple/'
```

## 4. 安装 openai 包

### 命令

```bash

```

### 安装了什么？

```
openai 1.x                  ← OpenAI API 的 Python SDK
```

```bash
pip list | grep openai
```

### 可选：指定版本

```bash

pip install openai>=1.0.0    # 安装最低版本
```

## 5. 安装 pipreqs

### 用来干什么？

```bash
# 传统做法：手动维护 requirements.txt
#   新增一个包 → 手动加一行 → 容易漏
#
# pipreqs 做法：自动扫描项目 import
#   import openai → 自动写入 requirements.txt
#   import flask  → 自动写入 requirements.txt
```

### 安装

```bash
pip install pipreqs
```

### 使用

```bash
# 扫描当前目录下的所有 .py 文件，生成 requirements.txt
pipreqs . --encoding=utf8 --force

# 参数说明：
#   .          → 扫描当前目录
#   --encoding → 指定编码（避免中文注释报错）
#   --force    → 覆盖已存在的 requirements.txt
```

### 生成的 requirements.txt

```
openai==1.x.x
flask==3.x.x
python-dotenv==x.x.x
...
```

## 6. 加载 .env 环境变量

### 为什么不用直接在代码里写死？

```python
# ❌ 致命错误：API Key 写死在代码里
openai.api_key = "sk-xxxxx"
# 1. 上传 GitHub 会泄露密钥
# 2. 不同环境要用不同 Key，改代码才能换
# 3. 团队协作时每个人都得改代码

# ✅ 正确做法：从环境变量读取
openai.api_key = os.getenv("OPENAI_API_KEY")
# 1. .env 被 .gitignore 忽略，永远不会上传
# 2. 不同环境不同 .env 文件
# 3. 只需复制 .env.example 改自己的值
```

### 项目结构

```
api/
├── .env                  ← 实际密钥（被 .gitignore 忽略）
├── .env.example          ← 密钥模板（提交到 Git）
└── .gitignore            ← 包含 .env
```

### .env 文件内容

```bash
# .env
OPENAI_API_KEY=sk-your-key-here
OPENAI_BASE_URL=https://api.openai.com/v1
FLASK_ENV=development
FLASK_PORT=5001
```

### .env.example（提交到 Git）

```bash
# .env.example（不填真实值）
OPENAI_API_KEY=your-key-here
OPENAI_BASE_URL=https://api.openai.com/v1
FLASK_ENV=development
FLASK_PORT=5001
```

### 加载 .env 的代码

```python
# 方式一：显式调用（推荐，可控性强）
from dotenv import load_dotenv
load_dotenv()  # 加载 .env 文件到 os.environ

# 方式二：放在 app/__init__.py 的 create_app() 中
# 这样每次启动都自动加载
```

### 验证加载成功

```python
import os
from dotenv import load_dotenv

load_dotenv()
print(os.getenv("OPENAI_API_KEY"))  # 应该输出你的 Key
```

### 关键原则

```
.env 文件 = 秘密保管箱
   ↓
load_dotenv() = 打开保管箱
   ↓
os.getenv() = 从保管箱取东西
   ↓
app.config  = 把取出来的东西放到 Flask 配置中心
```

## 7. 创建 Flask 应用实例

### Flask(**name**) 是什么？

```python
from flask import Flask
app = Flask(__name__)
```

前文已详细讲过，不重复。记住：

| 文件位置             | `__name__` 值 | Flask 算出的 root_path |
| -------------------- | ------------- | ---------------------- |
| `run.py`（直接运行） | `"__main__"`  | `run.py` 所在目录      |
| `app/__init__.py`    | `"app"`       | `app/` 的上一级        |

### 完整的最小实例

```python
# run.py —— 应用入口
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return {"message": "Hello World"}

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5001, debug=True)
```

启动：

```bash
python run.py
# 访问 http://localhost:5001 → {"message": "Hello World"}
```

### 应用工厂模式（进阶）

```python
# app/__init__.py
from flask import Flask

def create_app():
    app = Flask(__name__)

    # 加载配置
    app.config.from_object(...)

    # 注册扩展
    from flask_cors import CORS
    CORS(app)

    # 注册路由
    from app.routes.main import main_bp
    app.register_blueprint(main_bp)

    return app
```

```python
# run.py
from app import create_app
app = create_app()

if __name__ == '__main__':
    app.run()
```

## 8. 注册 Blueprint（路由模块）

### 为什么需要 Blueprint？

```
项目小的时候：
  @app.route('/hello')
  @app.route('/api/chat')
  @app.route('/api/users')
  → 全写在 run.py 里，还行

项目大了以后：
  run.py 3000 行，找路由要 Ctrl+F
  → 拆！按功能拆成模块
  → Blueprint 就是用来干这个的
```

### 类比

```
Flask app = 手机
Blueprint = App Store 里的 App

手机有了 App Store，才能装各种 App
Flask app 注册了 Blueprint，才能有各种路由
```

### 最短链路实现

**第 1 步：创建路由文件**

```python
# app/routes/main.py
from flask import Blueprint

# 创建 Blueprint 实例
main_bp = Blueprint('main', __name__)

@main_bp.route('/')
def index():
    return {"message": "Hello from Blueprint"}
```

| 参数       | 含义                                       |
| ---------- | ------------------------------------------ |
| `'main'`   | Blueprint 名称（路由反转时用）             |
| `__name__` | 和 Flask 一样，告诉 Blueprint 它在哪个模块 |

**第 2 步：在 create_app 中注册**

```python
# app/__init__.py
from flask import Flask

def create_app():
    app = Flask(__name__)

    # 注册 Blueprint
    from app.routes.main import main_bp
    app.register_blueprint(main_bp)  # ← 关键：这行把路由挂到 app

    return app
```

### 多 Blueprint 组织

```
app/
├── routes/
│   ├── __init__.py     ← 路由包
│   ├── main.py         ← 首页路由
│   ├── chat.py         ← 聊天路由
│   └── auth.py         ← 认证路由
└── __init__.py         ← create_app()
```

```python
# app/routes/chat.py
from flask import Blueprint

chat_bp = Blueprint('chat', __name__)

@chat_bp.route('/chat')
def chat():
    return {"message": "Chat endpoint"}
```

```python
# app/__init__.py
def create_app():
    app = Flask(__name__)

    from app.routes.main import main_bp
    from app.routes.chat import chat_bp
    from app.routes.auth import auth_bp

    app.register_blueprint(main_bp)
    app.register_blueprint(chat_bp)
    app.register_blueprint(auth_bp, url_prefix='/auth')
    #                                    ↑ 所有 auth 路由加 /auth 前缀

    return app
```

效果：

| Blueprint | 前缀    | 最终路由                        |
| --------- | ------- | ------------------------------- |
| `main_bp` | 无      | `/`                             |
| `chat_bp` | 无      | `/chat`                         |
| `auth_bp` | `/auth` | `/auth/login`、`/auth/register` |

## 9. 总结

整个环境搭建其实就这些步骤，回顾一下：

### 完整项目结构

```
api/
├── .venv/                    ← 虚拟环境（第1/2步）
├── .env                      ← 密钥配置（第6步）
│   ├── OPENAI_API_KEY=sk-xxx
│   ├── FLASK_ENV=development
│   └── FLASK_PORT=5001
├── .env.example              ← 密钥模板
├── .gitignore                ← 忽略 .env
├── requirements.txt          ← 依赖清单（第5步）
├── run.py                    ← 入口文件
├── app/
│   ├── __init__.py           ← create_app() 工厂
│   └── routes/
│       ├── __init__.py
│       └── main.py           ← Blueprint 路由
└── docs/
    └── ...                   ← 本文档
```

### 完整操作命令（按顺序执行）

```bash
# 1. 创建虚拟环境
python3 -m venv .venv

# 2. 激活虚拟环境
source .venv/bin/activate

# 3. 配置阿里云镜像（永久）
pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/

# 4. 安装核心依赖
pip install flask python-dotenv flask-cors openai

# 5. 安装 pipreqs（生成 requirements.txt）
pip install pipreqs
pipreqs . --encoding=utf8 --force
```

### 完整核心代码

```python
# run.py
from app import create_app

app = create_app()

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5001, debug=True)
```

```python
# app/__init__.py
import os
from dotenv import load_dotenv
from flask import Flask
from flask_cors import CORS

def create_app(config_name=None):
    # 1. 加载 .env
    load_dotenv()

    # 2. 创建 Flask 实例

    # 3. 注册 CORS
    CORS(app)

    # 4. 注册 Blueprint
    from app.routes.main import main_bp
    app.register_blueprint(main_bp)

    return app
```

```python
# app/routes/main.py
from flask import Blueprint

main_bp = Blueprint('main', __name__)

@main_bp.route('/')
def index():
    return {"message": "API is running"}
```
